Performing MCA using prince¶

In [11]:
import prince

# Step 1: Initialize the MCA Model
mca = prince.MCA(
    n_components=2,   # Number of dimensions to reduce to
    n_iter=3,          # Number of iterations for the algorithm
    copy=True,         # Copy the data internally (safer for modifications)
    check_input=True,  # Perform basic data checks
    engine='sklearn',  # Use scikit-learn's implementation (if available)
    random_state=42   # For reproducibility 
)

# Step 2: Fit the MCA Model
mca = mca.fit(dataset)  # Fits the model and performs one-hot encoding 

# Print eigenvalues and explained variance
print("\nEigenvalues:\n", mca.eigenvalues_) 
print("\nTotal Inertia:\n", mca.total_inertia_)
print("\nEigenvalues Summary:\n",mca.eigenvalues_summary)

Q: Explain the main parameters:

  • n_components: Specifies the number of dimensions (components) you want to reduce your data to. Lower values provide greater simplification.
  • n_iter: Determines how many times the MCA algorithm iterates to find the optimal solution.
  • copy: Making a copy of the data prevents accidental modifications to the original dataset.
  • check_input: This parameter performs checks to make sure your input data is appropriate for MCA (e.g., categorical data).
  • engine: If scikit-learn is installed, you can set this to 'sklearn' to potentially use its optimized MCA implementation.
  • random_state: This ensures that if you run the analysis multiple times, you'll get the same results. Important for reproducibility.

Q: How is the percentage of explained variance calculated for each component?

A: The percentage of explained variance for each component is calculated by dividing the component's eigenvalue by the total inertia: (Eigenvalue of component / Total Inertia) * 100

Q1: Explain the different mca.plot parameters:

  • x_component/y_component: Control which dimensions (components) are used for plotting on the x and y-axes of the factor map. show_column_markers/show_row_markers: Decide whether to display visual points for column modalities or individual rows.
  • show_column_labels/show_row_labels: Toggle the display of labels for modalities (often clearer) or rows.

Q2: What do these factor maps show?

Answer: Factor maps visually represent how categories of different variables relate to each other in the reduced dimensional space:

  • Close Proximity: Modalities or individuals close together share similarities in the analyzed variables.
  • Opposing Directions: Points or groups located on opposite sides of the map likely represent contrasting characteristics or behaviors.

Conclusion and Q&A¶

  • Dimension 1 (Component 0): Explains 40.17% of the total variance in the dataset. This suggests that this dimension captures a substantial portion of the variability in the categorical variables.

  • Dimension 2 (Component 1): Explains 21.11% of the total variance. While less influential than Dimension 1, it still contributes meaningfully to understanding the overall variability.

Combined, the first two dimensions account for 61.28% of the total variance. This indicates a reasonably good representation of the original data in a reduced two-dimensional space. However, it's important to note that around 38.72% of the information is not captured, so some relationships might be lost in this simplification.

  • Column Contributions (Modalities):

    • Inflated_T: Has the highest contribution to Dimension 1. This implies that whether a balloon is inflated (True) or not (False) is a dominant factor influencing the position of data points along this dimension.

    • Action_DIP: Also contributes notably to Dimension 1, suggesting that the "DIP" action plays a role in differentiating data points.

    • Color_PURPLE and Color_YELLOW: Have very low contributions to both dimensions, indicating that these colors might not be strong differentiators in the context of the other variables.

  • Row Contributions (Individuals):

Rows 3, 4, 14, and 15: Show high contributions to Dimension 1, meaning these specific combinations of balloon characteristics are influential in defining that dimension.

  • Cosine Similarities:

    • Rows: High cosine similarities for certain row pairs (e.g., 14 and 15) on Dimension 1 suggest these rows have similar profiles across the categorical variables.

    • Columns: "Inflated_T" and "Inflated_F" likely have a strong negative correlation on Dimension 1, as they are the main contributors but with opposite meanings.

Overall Interpretation:

The MCA results suggest a relationship between a balloon being inflated ("Inflated_T") and potentially other features like "Action_DIP." Rows 3, 4, 14, and 15 likely represent balloon characteristic combinations strongly associated with either inflated or deflated outcomes.
